PRIMARY TRACK: FORTE ECOSYSTEM
================================================================================

REQUIREMENT: Deep integration with Forte Actions and Scheduled Transactions

✅ IMPLEMENTATION:
------------------

1. **Forte Actions Metadata (3 Actions)**
   
   a) Stake and Schedule Settlement
      - File: contracts/forte-actions/stake-and-schedule.json
      - Composite action combining staking + settlement scheduling
      - Safety metadata: max stakes, token whitelist, confirmations
      - Execution: bash scripts/run_forte_action.sh
      
   b) Request AI Lineup
      - File: contracts/forte-actions/request-ai-lineup.json
      - AI-powered lineup optimization action
      - Rate limiting: 10 requests/hour per user
      - Response schema with expected score and confidence
      
   c) Create League
      - File: contracts/forte-actions/create-league.json
      - On-chain league creation with validation
      - Configurable parameters and constraints
      - Creator deposit requirement

2. **Scheduled Transactions Implementation**
   
   Contract: contracts/Settlement.cdc
   - scheduleSettlement() function (line 90)
   - Stores scheduled time and TX ID
   - Automatic execution at league end time
   - Emits SettlementScheduled event
   
   Transaction: transactions/schedule_settlement.cdc
   - Accepts leagueId, scheduledTime, txId
   - Updates league state with settlement info
   - Links to Forte scheduler

3. **Safety Metadata**
   
   All actions include:
   - maxStakePerUser: 10,000 FLOW
   - allowedTokens: [FLOW, FUSD, USDC]
   - requiresConfirmation: true
   - minimumConfirmations: 1
   - reentrancyProtection: true
   - escrowValidation: true

4. **Composability**
   
   Actions can be chained:
   - create-league → stake-and-schedule
   - request-ai-lineup → submit-lineup → stake-and-schedule
   
   Data passing between actions:
   - leagueId from create → stake
   - lineup from AI → submit
   - txId from stake → schedule

TRACK: AI/ML INNOVATION
================================================================================

REQUIREMENT: Incorporate AI/ML for enhanced user experience

✅ IMPLEMENTATION:
------------------

1. **AI Lineup Predictor**
   
   File: ai/app.py
   Class: LineupPredictor
   
   Algorithm:
   ```
   score = α * recent_performance (45%)
         + β * market_value (30%)
         + γ * consistency (15%)
         + δ * trending (10%)
         - injury_risk
   ```

2. **Multiple Strategies**
   
   - Balanced: Equal weight distribution
   - High-Risk: Favor high market value (1.25x multiplier)
   - Conservative: Favor consistency + low injury (1.15x multiplier)

3. **Explainable AI**
   
   Every prediction includes:
   - Position assignments with reasoning
   - Expected score calculation
   - Confidence level (0-1)
   - Human-readable rationale
   
   Example rationale:
   "Optimized for balanced strategy. Expected score: 387.5. 
    3 players trending upward. High-value player concentration."

4. **API Integration**
   
   Endpoint: POST /api/ai/predict-lineup
   Response time: <3 seconds
   Rate limiting: 10 requests/hour per user
   Fallback: Random lineup if AI service down

5. **Data Sources**
   
   Current (mock):
   - Player performance (0-100 scale)
   - NFT market value ($50-$2000)
   - Consistency (0-1)
   - Injury risk (0-1)
   - Trending (up/stable/down)
   
   Future (production):
   - Find Labs API for on-chain data
   - Dapper Moments API for NFT metadata
   - ESPN/ChainLink for real sports data
